# Process Improvements Checklist

Purpose: Standard checklist to validate process changes and documentation updates.

- [ ] Problem or gap clearly described
- [ ] Impact and stakeholders identified
- [ ] Proposed change (copy or diff) provided
- [ ] Acceptance criteria defined (measurable where possible)
- [ ] Cross-team review performed (list reviewers)
- [ ] Update applied to docs/ and referenced in project README
- [ ] Confirm communication plan for affected teams
- [ ] Follow-up review scheduled to measure impact
